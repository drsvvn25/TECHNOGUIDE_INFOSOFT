NLP Text Classification Theory

1. Text Preprocessing

Computers do not natively understand English text. The raw string must be cleaned and broken down. Preprocessing reduces the vocabulary dimensionality, merging words that mean the same thing and stripping details that have no statistical importance (like punctuation and general filler words like 'is', 'the', and 'and', known as stopwords).

2. Feature Extraction (CountVectorizer)

To train a Machine Learning model, we must convert the cleaned text tokens into numbers. This is done using the Bag of Words model. The CountVectorizer calculates the vocabulary of all words in the training dataset, then creates a matrix where each row is a message and each column represents a word from the vocabulary. The cells represent how many times that word appears in the message.

3. Multinomial Naive Bayes Classifier

The Multinomial Naive Bayes classifier is highly suited for document classification. It works using Bayes' Theorem of conditional probability:

P(Class | Word) = [ P(Word | Class) × P(Class) ] / P(Word)

It is called "Naive" because it assumes that the occurrence of each word is independent of all other words. Despite this simple assumption, Naive Bayes is extremely effective, fast, and works exceptionally well for text applications like spam filters or sentiment analyzers.

Pipeline Summary

  • 1
    Raw Text Clean-up: Lowercasing letters and removing punctuation (string.punctuation).
  • 2
    Tokenization & Stopwords: Splitting into tokens and filtering out words from NLTK's English stopword dictionary.
  • 3
    Vectorization: Using CountVectorizer to construct a document-term frequency matrix.
  • 4
    Training Split: Splitting dataset (e.g. 70/30) to compute accuracy on unseen messages.
  • 5
    Model Learning: Fitting a MultinomialNB probability model based on word likelihoods.
  • 6
    Active Prediction: Computing category probabilities (predict_proba) for new user queries.
